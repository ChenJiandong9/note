# 考研数学复习笔记项目 - Gemini CLI 指南

本文件定义了针对此项目的特定指令和规范，优先于通用系统指令。

## 项目核心

- **目标**：系统化总结、复习和管理考研数学（高等数学）知识点。
- **技术栈**：Obsidian (Markdown), LaTeX, Git.
- **核心目录**：`高数/` (主要工作区，重要度 ⭐⭐⭐)。
- **参考目录**：`高数资料/` (仅作参考，除非明确要求，否则不主动查阅，禁止修改)。

## 视觉与格式规范 (必须严格遵守)

### 0. 元数据与标签规范 (New)
每篇笔记必须包含以下 YAML 区块：
```markdown
---
tags:
  - 高数/模块名/具体章节
  - 类型/内容性质 (概念/计算/应用/技巧)
aliases:
  - 别名 (如：第X讲)
---
```
**推荐标签：**
- `#高数/极限`
- `#高数/一元微分`
- `#高数/一元积分`
- `#高数/多元微积分`
- `#高数/微分方程`
- `#类型/概念` | `#类型/计算` | `#类型/应用` | `#类型/技巧`

### 1. Admonition 提示框 (Obsidian 插件)

必须使用以下语法为不同类型的内容添加提示框：

| 类型 | 语法 | 用途 |
| :--- | :--- | :--- |
| **Danger** | `> [!danger]` | **核心内容**：严格定义、重要公式、考试必备 |
| **Warning** | `> [!warning]` | **重要内容**：需要掌握的性质、分类 |
| **Important** | `> [!important]` | **关键原则**：重要性质、准则 |
| **Tip** | `> [!tip]` | **技巧方法**：解题技巧、口诀、特殊方法 |
| **Note** | `> [!note]` | **基础概念**：一般定义、背景知识 |
| **Info** | `> [!info]` | **补充说明**：常用特例、一般信息 |

### 2. 数学公式 (LaTeX)

- **行内公式**：`$公式$`
- **块级公式**：
  ```latex
  $$
  公式内容
  $$
  ```
- **注意**：在提示框内使用 `$$` 也能正确渲染。

### 3. 表格与总结

- 凡是涉及分类、对比、等价替换表的内容，必须使用 Markdown 表格。
- 重点内容在表格中建议加粗。

## Gemini CLI 工作准则

1. **专注区域**：所有新笔记和修改应发生在 `高数/` 目录下。
2. **数学准确性**：在输出公式或定义前，必须进行双重检查，确保符合数学严谨性。
3. **保持风格一致**：严格遵循上述提示框颜色约定，不要随意更改。
4. **知识链接**：主动建立笔记间的 `[[双向链接]]`。
5. **严禁修改资料区**：禁止对 `高数资料/` 目录进行任何写操作。

## 特定工作流

### 1. 问答归档模式 (Inquiry & Archiving)

- **互动阶段**：用户开启提问模式，Gemini 提供解答。
- **实时记录**：所有本轮对话的问答内容将实时同步/追加到 `question/answer.md` 中，方便用户即时浏览历史。
- **归档触发**：当用户发出“记录”、“归档”或“总结总结”等指令时。
- **执行归档**：
    1. 从 `answer.md` 中提炼内容，在 `question/` 目录下新建 `00X-主题.md` 文件。
    2. 归档完成后，根据用户需要保留或清理 `answer.md`。
    3. 规范：归档文件必须严格遵守 **视觉与格式规范**。

## 常用命令参考

- **搜索知识点**：`grep_search(pattern="关键词", dir_path="高数")`
- **新建笔记**：`write_file(file_path="高数/新主题.md", content="...")`
- **检查链接**：`glob(pattern="高数/**/*.md")`
